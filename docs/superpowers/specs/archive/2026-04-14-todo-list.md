## 一、声音系统说明

### 当前实现

声音系统**已默认使用 macOS 系统内置声音**，无需额外配置即可正常工作：

| 事件类型 | 系统声音名称 | 说明 |
|---------|-------------|------|
| 权限请求 (`permissionRequest`) | `Glass` | 清脆的玻璃声，提醒用户注意 |
| 任务完成 (`completed`) | `Hero` | 英雄音效，表示任务完成 |
| 错误 (`error`) | `Basso` | 低沉的提示音，表示出错 |
| 上下文压缩 (`compacting`) | `Pop` | 轻柔的弹出声，表示压缩中 |

这些声音名称是 `NSSound` 内置的系统声音，macOS 自带，无需任何额外文件。

from PIL import Image

    img = Image.new('RGBA', (size, size), (40, 160, 220, 255))  # 蓝色背景

## 三、OpenCode 插件详解

### 插件的作用

OpenCode 插件（`vibe-island.js`）是 **Vibe Island 监控 OpenCode 的核心组件**，作用类似于 Claude Code 的 Hook 系统。

#### 工作原理

```
OpenCode 运行
    ↓
加载 vibe-island.js 插件（进程内运行）
    ↓
订阅 OpenCode 事件（session.created, tool.execute.before, permission.ask 等）
    ↓
事件触发时，调用 vibe-island CLI（execFileSync）
    ↓
传入 JSON 数据（通过 stdin）
    ↓
CLI 写入 ~/.vibe-island/opencode-sessions/<pid>.json
    ↓
SessionFileWatcher 检测到文件变化
    ↓
更新灵动岛状态
```

#### 插件做了什么

1. **事件订阅** — 监听 OpenCode 的 10 种事件：
   - `session.created` → SessionStart
   - `session.idle` → Stop
   - `session.error` → SessionError
   - `session.compacted` → PostCompact
   - `chat.message` → UserPromptSubmit
   - `tool.execute.before` → PreToolUse
   - `tool.execute.after` → PostToolUse
   - `permission.ask` → PermissionRequest
   - `experimental.session.compacting` → PreCompact

2. **数据规范化** — 将 OpenCode 的事件格式转换为 Vibe Island 统一的 `SessionEvent` 格式：
   - 工具名称映射：`bash` → `Bash`，`read` → `Read`（小写转 PascalCase）
   - 字段名映射：`filePath` → `file_path`（camelCase 转 snake_case）

3. **调用外部 CLI** — 使用 `execFileSync` 同步调用 `vibe-island hook <EventType>`：
   - 5 秒超时
   - 错误静默捕获（不影响 OpenCode 正常运行）
   - JSON 通过 stdin 传入

#### 为什么需要插件

OpenCode **不像 Claude Code 那样提供原生的 Hook 配置**。Claude Code 允许在 `~/.claude/settings.json` 中声明式配置 hook，而 OpenCode 需要通过插件系统来扩展功能。

插件是 OpenCode 四级降级架构中的 **L1（最高优先级）**：
- L1: **Plugin Hook** ← 插件实现，延迟 <100ms
- L2: SSE 事件流 ← 需要 OpenCode 以 serve 模式运行
- L3: 文件监控 ← 轮询，延迟 2s
- L4: 进程检测 ← 仅检测是否存在

**没有插件，OpenCode 只能用 L2-L4 降级方案**，实时性和状态精确度会降低。

#### 插件部署位置

```
~/.config/opencode/plugins/vibe-island.js
```

OpenCode 启动时会自动加载该目录下的所有 `.js` 文件作为插件。
