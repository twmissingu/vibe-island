# Vibe Island 项目 - 技术不确定性评估报告

> 评估日期：2026-04-13
> 评估目标：识别仍存在不确定性的技术点，评估风险等级

---

## 核心结论

**总体技术可行性：高（78% 可复用），但仍有 9 个技术点存在不同程度的不确定性。**

---

## 一、高不确定性技术点（需重点验证）🔴

### 1. OpenCode Plugin Hook 实际可用性

**不确定性等级**：🔴 高

**问题描述**：
- cctop 项目的插件代码是 **推断实现**，未验证其完整性和实际运行效果
- OpenCode 插件 API 的 `input.event.subscribe()` 签名和事件格式 **未官方文档化**
- 插件调用外部命令（`await $\``vibe-island hook ...`）的性能和可靠性 **未验证**
- cctop 项目（70⭐）的插件源码 **未实际阅读**，仅基于架构描述推断

**风险影响**：
- 如果插件 API 不稳定或事件格式与预期不同，Level 1 方案可能失效
- 用户需手动安装插件，安装率可能很低

**验证计划**：
- [ ] 克隆 cctop 项目，阅读实际插件源码
- [ ] 本地安装 OpenCode + cctop 插件，验证实际运行
- [ ] 测试插件事件订阅的完整流程
- [ ] 验证插件调用外部命令的性能和可靠性

**降级方案**：已有（SSE → 文件监控 → 进程检测）

---

### 2. OpenCode Session 文件格式

**不确定性等级**：🔴 高

**问题描述**：
- 文档标注"需要自行调研"，但实际 **未验证** session 文件的真实格式
- `~/.local/share/opencode/storage/` 目录结构是 **推断的**，可能与实际不同
- 文件是 JSON 还是 JSONL 或其他格式（如 SQLite）**未确认**
- 字段名称和嵌套结构 **完全未知**

**风险影响**：
- Level 3 文件监控方案可能无法正确解析
- 如果格式频繁变化，维护成本高

**验证计划**：
- [ ] 实际运行 OpenCode，检查 `~/.local/share/opencode/` 目录结构
- [ ] 读取 session 文件，确认格式和字段
- [ ] 对比 OpenCode 版本升级前后的格式变化
- [ ] 测试文件写入延迟（从事件发生到文件更新的时延）

**降级方案**：Level 4 进程检测（仅显示基础状态）

---

### 3. OpenCode SSE 事件格式和稳定性

**不确定性等级**：🔴 高

**问题描述**：
- SSE 事件格式基于 **博客文章推断**，未阅读官方文档或 SDK 源码
- `/global/event` 和 `/event` 端点的实际返回格式 **未验证**
- Issue #6447 报告 `/session/status` 曾返回空，说明 **API 不稳定**
- SSE 连接的心跳机制、超时处理、断线重连策略 **未明确**
- 官方 SDK（TypeScript/Python）的事件解析逻辑 **未实际阅读**

**风险影响**：
- Level 2 方案可能在 OpenCode 版本升级后失效
- 解析错误可能导致事件丢失

**验证计划**：
- [ ] 启动 `opencode serve`，实际连接 SSE 端点
- [ ] 捕获并分析实际事件流的 JSON 格式
- [ ] 阅读 OpenCode SDK 源码，确认事件 Schema
- [ ] 测试 SSE 连接的稳定性和断线重连

---

### 4. 像素宠物 Swift 渲染实现

**不确定性等级**：🔴 高

**问题描述**：
- claude-buddy 是 **Electron + JavaScript** 实现，渲染逻辑不能直接复用
- 需要将 hex 帧数据转换为 **SwiftUI Canvas 渲染**，这是从零实现
- 14 款宠物 × 3 状态 × 2 帧 = 84 个帧数据的 **实际可用性未验证**
- claude-buddy 项目仅 7⭐，帧数据可能 **不完整或有版权风险**

**风险影响**：
- 如果 Swift Canvas 渲染性能不佳，可能导致卡顿
- 帧数据如果缺失或有版权问题，需要重新设计

**验证计划**：
- [ ] 阅读 claude-buddy 的 pets.js 源码，确认帧数据格式
- [ ] 实现 Swift Canvas 渲染原型，测试性能
- [ ] 验证 84 个帧数据的完整性和质量
- [ ] 考虑设计专属像素宠物（备选方案）

## 二、中不确定性技术点（需测试验证）🟡

### 5. Claude Code Hook stdin JSON 实际格式

**不确定性等级**：🟡 中

**问题描述**：
- 12 种事件的字段定义基于 **文档和参考项目推断**
- cc-status-bar 的 `HookEvent.swift` 字段名可能与实际 stdin JSON **不完全匹配**
- 某些事件（如 SubagentStart/Stop）的字段结构 **未实际捕获验证**
- Claude Code 版本升级后，hook 格式可能 **向后不兼容**

**风险影响**：
- JSON 解析失败会导致事件丢失
- 字段缺失可能导致状态判断错误

**验证计划**：
- [ ] 实际运行 Claude Code hook，捕获 stdin JSON
- [ ] 验证 12 种事件的完整字段
- [ ] 测试版本升级后的兼容性
- [ ] 添加容错解析（缺失字段使用默认值）

---

### 6. DispatchSource 文件监听可靠性

**不确定性等级**：🟡 中

**问题描述**：
- DispatchSourceFileSystemObject 在 **高频写入场景** 下的表现未验证
- 文件描述符泄漏、事件丢失、延迟触发等边界情况 **未测试**
- cc-status-bar 的防抖动逻辑（debounce）的参数设置 **未调优**
- macOS 沙盒权限对文件监听的影响 **未验证**

**风险影响**：
- 事件丢失会导致 UI 状态不同步
- 频繁触发可能导致性能问题

**验证计划**：
- [ ] 实现 DispatchSource 监听原型
- [ ] 高频写入测试（模拟 10 个并发会话）
- [ ] 验证防抖动参数的最佳值
- [ ] 测试沙盒权限下的行为

---

### 7. NSPanel 多桌面/全屏/ Mission Control 兼容性

**不确定性等级**：🟡 中

**问题描述**：
- Lyrisland 的 NSPanel 配置（`.canJoinAllSpaces`, `.stationary`）在 **不同 macOS 版本** 的行为可能不同
- 全屏应用切换、Mission Control、Expose 等场景下的表现 **未验证**
- 多显示器场景下的位置记忆 **可能有问题**
- 与 macOS 原生通知、控制中心的 **交互冲突未测试**

**风险影响**：
- 特定场景下窗口可能异常隐藏或位置错乱
- 用户体验不一致

**验证计划**：
- [ ] 在 macOS 14/15 多版本测试
- [ ] 全屏应用切换测试
- [ ] Mission Control / Expose 测试
- [ ] 多显示器场景测试

---

### 8. Hook 自动安装的兼容性和安全性

**不确定性等级**：🟡 中

**问题描述**：
- cc-status-bar 的 SetupManager 逻辑是 **参考实现**，非直接复用
- 非破坏性更新 `~/.claude/settings.json` 的 JSON 合并逻辑 **复杂度高**
- 用户手动修改 settings.json 后的 **冲突处理未明确**
- macOS 权限（自动化权限、文件访问权限）对自动安装的 **影响未验证**

**风险影响**：
- 错误的 JSON 修改可能导致 Claude Code 配置损坏
- 用户可能不信任自动修改配置文件的行为

**验证计划**：
- [ ] 实现 SetupManager 原型
- [ ] 测试各种 settings.json 初始状态（空、有 hooks、有其他工具 hooks）
- [ ] 测试冲突场景（用户手动修改 + 自动更新并发）
- [ ] 添加完整的备份和回滚机制

**风险影响**：低（即使失败，仅影响 Codex 监控）

**验证计划**：
- [ ] 实际运行 Codex，验证进程名称和 pgrep 匹配
- [ ] 测试 cwd 匹配的准确性
